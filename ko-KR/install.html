Rust 설치에 대한 참고 사항

rustup으로 툴체인 관리하기

Rust는 rustup 툴로 설치되고 관리됩니다. Rust는 6주 간격의 빠른 릴리스 프로세스를 가지고 있으며 많은 수의 플랫폼을 지원하기 때문에, 언제나 수많은 Rust 빌드가 존재합니다. rustup은 Rust가 지원하는 모든 플랫폼에서 이 빌드들을 일관되게 다루며, 베타 및 나이틀리 릴리스 채널에서 Rust를 설치하는 걸 가능케 하고, 추가적인 크로스컴파일 타겟 또한 지원합니다.

더 자세한 정보는 rustup 문서를 참고하세요.

PATH 환경 변수 설정하기

Rust 개발 환경에서 모든 도구는 ~/.cargo/bin %USERPROFILE%\.cargo\bin 디렉토리에 설치되며, rustc, cargo, 그리고 rustup을 포함한 Rust 툴체인은 여기에서 찾을 수 있습니다.

따라서 Rust 개발자는 이 디렉토리를 PATH 환경 변수에 추가하는 게 일반적입니다. 설치 과정에서 rustup은 PATH를 설정하려 시도하나, 플랫폼의 차이, 명령줄 셸의 차이 및 rustup의 버그 때문에 PATH가 콘솔이 재시작될 때까지, 또는 사용자가 로그아웃할 때까지, 또는 아예 변경되지 않을 수도 있습니다.

만약 설치 후에 콘솔에서 rustc --version를 실행했을 때 실패한다면 이게 가장 가능성이 높은 이유입니다.

윈도 한정

윈도에서 Rust는 추가적으로 Visual Studio 2013 또는 이후를 위한 C++ 빌드 도구를 필요로 합니다. 빌드 도구를 받는 가장 쉬운 방법은 Microsoft Visual C++ Build Tools 2017을 설치하여 Visual C++ 빌드 도구만을 설치하는 것입니다. 아니면 Visual Studio 2017, Visual Studio 2015나 Visual Studio 2013을 설치한 뒤 설치 과정에서 "C++ 도구"를 선택해도 됩니다.

윈도에서 Rust를 설정하는 데 더 자세한 정보는 윈도용 rustup 문서를 참고하십시오.
